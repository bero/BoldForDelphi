LogBridge

Overview

The LogBridge example demonstrates how to integrate Bold's internal logging system with your application's logging infrastructure.

Bold uses its own logging mechanism through BoldLog calls throughout the framework. This example shows how to capture those log messages and redirect them to a file using TBoldFileLogSink.

Key Concepts

BoldLogManager

The central registry for log sinks. All log messages pass through the BoldLogManager which distributes them to registered sinks.

TBoldFileLogSink

A log sink that writes messages to a file with timestamps and log level indicators. Each log entry includes:
  • Timestamp (date and time with milliseconds)
  • Log level: [INFO], [DEBUG], [WARN], [ERROR]
  • The log message

Log Levels

Bold supports multiple log levels that can be filtered:
  • llInfo - General information messages
  • llDebug - Debug-level details
  • llWarning - Warning messages
  • llError - Error messages
Use AllBoldLogLevels to receive all levels, or specify a set like [llError, llWarning] to filter.

Usage

var
  FileSink: TBoldFileLogSink;
begin
  // Create and register a file sink
  FileSink := TBoldFileLogSink.Create('application.log');
  FileSink.Levels := AllBoldLogLevels;
  BoldLogManager.RegisterSink(FileSink);

  // Now all BoldLog calls will write to the file
  BoldLog('Application started');
  BoldLog('Loading data...', llDebug);
end;

Example Output

2025-12-18 20:38:59.139 [INFO] Application started
2025-12-18 20:38:59.140 [DEBUG] Loading data...
