Master Detail

Display parent-child relationships with linked grids

Overview

This example demonstrates how to display master-detail relationships using Bold handles and grids. It shows Projects (master) and their related Tasks (detail) with automatic synchronization when the selection changes.

The example uses the shared Demo model with Project and Task classes, demonstrating how to link TBoldListHandle components to create cascading views.

How to Run

Manage Projects and their Tasks using the linked grids:

  1. Open the system - Click "Open system" to connect to the database
  2. View projects - The Projects grid shows all projects in the system
  3. Select a project - Click a project row to see its tasks in the Tasks grid
  4. Add projects - Click "Add" to create a new project
  5. Add tasks - Use the navigator on the Tasks grid to add tasks to the selected project
  6. Edit in-place - Double-click cells to edit values directly
  7. Delete projects - Select a project and click "Delete" (confirms before deleting)
  8. Save changes - Click "Save" to persist all changes to the database

Features Demonstrated

Feature Implementation
Master list showing all projects TBoldListHandle with Project.allInstances
Detail list filtered by selection TBoldListHandle.RootHandle pointing to master
Automatic detail refresh on selection change Bold subscription mechanism
Cascading delete with confirmation TProject.Delete removes related tasks
Dirty object tracking System.DirtyObjects.Count check on close
Shared model and database DemoDataModule / DemoClasses

Implementation Details

Master Handle Setup

The master TBoldListHandle retrieves all Project instances: 

lhaProjects.Expression := 'Project.allInstances';

Detail Handle Setup

The detail handle uses the master handle as its root and navigates the tasks association: 

lhaProjectTasks.RootHandle := lhaProjects;
lhaProjectTasks.Expression := 'tasks';

Shared Data Module

This example uses the shared DemoDataModule which provides database connectivity and the Bold system handle. The model includes Person, Project, Task, Building, and Ownership classes.
