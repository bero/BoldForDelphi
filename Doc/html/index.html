What is Bold for Delphi?

Bold for Delphi is a Model-Driven Architecture (MDA) framework and Object-Relational Mapping (ORM) tool that lets you design your application's domain model in UML and automatically generates both Delphi classes and database schemas from it. Originally developed by Boldsoft and later owned by Borland, it was open-sourced by Embarcadero in 2020 and is now community-maintained.

Unlike simpler ORMs that just map objects to tables, Bold provides a complete object space — a runtime environment where your business objects live, interact, and automatically stay synchronized with both the database and the user interface.

Why Use Bold?

The Problem Bold Solves

Every data-driven Delphi application faces the same challenges: How do you persist objects to a database? How do you keep UI controls in sync when data changes? How do you implement calculated fields that update automatically? How do you handle undo/redo? How do you validate business rules consistently?

Traditional approaches require writing large amounts of boilerplate code: SQL statements, dataset-to-object mapping, manual UI refresh logic, change tracking, and validation code scattered throughout the application. This code is tedious, error-prone, and difficult to maintain as requirements evolve.

Bold's Approach

Bold eliminates this boilerplate by providing a declarative, model-driven approach:

  • Design your domain model visually using UML classes, attributes, and associations
  • Define business rules with OCL (Object Constraint Language) — a powerful query language for objects
  • Generate code automatically — Bold creates strongly-typed Delphi classes from your model
  • Bind UI controls directly to objects — changes propagate automatically in both directions
  • Let Bold handle persistence — objects are transparently loaded and saved to the database

Use Cases

Enterprise Business Applications

ERP, CRM, and line-of-business applications with complex domain models benefit from Bold's model-driven approach. Define entities like Customer, Order, Invoice with their relationships once, and Bold handles the rest.

Data-Centric Desktop Applications

Applications that display, edit, and report on structured data. Bold's automatic UI synchronization means editing an object in one form instantly updates all other views showing the same data.

Applications with Complex Business Rules

When you have calculated fields, validation rules, and constraints that span multiple objects, Bold's derived attributes and OCL constraints express these declaratively rather than imperatively.

Multi-User Database Applications

Bold's optimistic locking, object identity management, and transaction support make it straightforward to build applications where multiple users work with shared data.

Advantages Over Other Approaches

vs. Raw SQL / TDataSet

  • Type safety — Work with strongly-typed objects (Customer.Name) instead of string-based field access (DataSet.FieldByName('Name'))
  • Object identity — Each database row maps to exactly one object instance; the same customer loaded twice returns the same object
  • Automatic change tracking — Bold knows what changed and generates minimal UPDATE statements
  • Relationship navigation — Access related objects naturally (Order.Customer.Address) without manual joins

vs. Simple ORMs

  • Derived attributes — Define calculated properties (like Order.TotalPrice) that automatically recalculate when dependencies change
  • Automatic UI binding — Bold-aware controls subscribe to objects and update automatically; no manual refresh code
  • OCL queries — Query objects using a powerful, standardized language: Person.allInstances->select(age > 30)
  • Built-in undo/redo — Full transaction support with rollback to any checkpoint
  • Model evolution — Change your UML model and Bold updates the database schema automatically

vs. Hand-Coded Business Objects

  • Code generation — Business classes are generated from your UML model, eliminating boilerplate
  • Consistency — All objects follow the same patterns for persistence, validation, and change notification
  • Maintainability — Changes to the model automatically propagate to generated code

Key Features

Model-Driven Development

Design your domain in UML, Bold generates Delphi classes with full persistence support

Automatic UI Synchronization

Changes to objects instantly update all bound UI controls - no manual refresh needed

OCL Expression Language

Powerful Object Constraint Language for queries, derived values, and business rules

Derived Attributes

Define calculated properties that automatically update when dependencies change

Built-in Undo/Redo

Full transaction support with unlimited undo levels and checkpoint management

Object Identity

Each database row maps to exactly one object instance - guaranteed consistency

Optimistic Locking

Safe multi-user access with automatic conflict detection and resolution

Multi-Database Support

Works with InterBase, MS SQL, Oracle, PostgreSQL, SQLite and more
