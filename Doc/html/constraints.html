Constraints

Define and validate business rules using OCL

Overview

Constraints define business rules that must be satisfied. Bold uses OCL expressions to define constraints that are automatically validated before saving.

Constraint Types

  • Attribute constraints - Value must be within range
  • Relationship constraints - Cardinality rules
  • Cross-attribute - End date must be after start date
  • Invariants - Conditions that must always be true

How Constraints Work

Before saving, Bold validates all constraints. Objects with violations cannot be persisted:

OCL Examples

ConstraintOCL Expression
Age must be positiveself.age >= 0
Name requiredself.name.size > 0
Valid date rangeself.endDate > self.startDate
Max 5 itemsself.items->size <= 5
