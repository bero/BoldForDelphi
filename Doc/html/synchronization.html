Synchronization

Keep multiple ObjectSpaces in sync

Overview

ObjectSpace synchronization allows multiple running instances to stay synchronized when one instance makes changes to shared data.

Use Cases

  • Multi-user applications sharing a database
  • Client-server architectures
  • Real-time collaboration features

How It Works

How to Test

  1. Run multiple instances of the application
  2. In one instance, create or modify an object
  3. Save changes to the database
  4. Watch the other instances update automatically
