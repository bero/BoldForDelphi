Renderers

Custom display and editing logic for Bold components

Overview

Renderers provide a powerful way to customize how Bold displays and handles data. They can transform values for display, apply conditional formatting, and even implement reverse derivation for writable calculated fields.

Renderer Types

TBoldAsStringRenderer

Convert any value to string with custom formatting, colors, and fonts

TBoldAsCheckBoxStateRenderer

Display boolean or calculated values as checkboxes

TBoldAsIntegerRenderer

Custom integer formatting and validation

TBoldAsFloatRenderer

Floating-point display with precision control

How Renderers Work

Renderers transform data between the model and UI, controlling display and input:

Common Use Cases

  • Conditional formatting - Red for negative values, green for positive
  • Calculated display - Show derived values without model changes
  • Status indicators - Convert enums to meaningful icons or text
  • Reverse derivation - Allow editing of calculated values
  • Validation feedback - Visual cues for invalid data

Key Events

Event Purpose
OnGetAsString Transform value for display
OnSetAsString Parse user input back to value (reverse derivation)
OnSetFont Customize font based on value
OnSetColor Customize colors based on value
OnSubscribe Set up dependencies for automatic refresh
