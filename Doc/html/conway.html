Conway's Game of Life

Cellular automaton implemented with Bold

Overview

This example implements Conway's Game of Life using Bold's object model. It demonstrates how Bold can handle complex object interactions and automatic UI updates based on model changes.

The Game of Life is a cellular automaton where cells live or die based on the number of neighboring cells, creating fascinating patterns from simple rules.

Bold Features Demonstrated

  • Object relationships - Each cell knows its neighbors
  • Derived attributes - Neighbor count calculated automatically
  • Batch updates - Efficient processing of generation steps
  • UI binding - Grid display updates automatically

How to Run

  1. Click cells to toggle them alive/dead (create initial pattern)
  2. Click Step to advance one generation
  3. Click Run for continuous animation
  4. Try classic patterns: glider, blinker, beacon, glider gun
