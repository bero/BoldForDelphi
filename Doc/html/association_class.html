Association Class

Add attributes to many-to-many relationships

Overview

Association classes allow you to add attributes to relationships. For example, an Ownership association between Person and Building can have a "share percentage" attribute.

Use Cases

  • Ownership - Person owns Building with share percentage
  • Employment - Person works at Company with start date and role
  • Enrollment - Student takes Course with grade

UML Diagram

An association class adds attributes to a many-to-many relationship:

How It Works

Without an association class, you can only link Person to Building. With the Ownership association class, each link also stores:

  • What percentage of the building this person owns
  • When they acquired the ownership
  • Whether they are the primary owner

Bold automatically creates and manages these association objects when you link persons to buildings.
