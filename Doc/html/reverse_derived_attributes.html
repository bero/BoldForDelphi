Reverse Derived Attributes

Make derived attributes writable

Overview

A derived attribute is a value calculated from other values. For many derived attributes, reversing the derivation doesn't make sense - what would it mean to change an invoice total? How would you apply it to the invoice items?

However, for some derived attributes the reverse is straightforward. Consider age and birth date: it's obvious how to update age if birthday changes and vice versa. This example demonstrates how to implement such reverse derived attributes.

How It Works

Reverse derivation allows editing a calculated value and propagating changes back to source attributes:

How to Run

  1. Add a person using the navigator. Enter a full name in the format firstname lastname
  2. Leave the column - watch how the full name is separated into firstname and lastname. The lastname is stored in a related TFamily object
  3. Add another person with the same lastname - no new family object is created
  4. Change the lastname in fullName for one person:
    • If multiple people share that family, a new family object is created
    • If only one person in the family, the familyname is updated

Implementation

_fullName_ReverseDerive

Whenever the full name changes, TPerson._fullName_ReverseDerive executes. This method handles what it means to change the name - manipulating not only attribute values, but also creation/deletion of objects and setting of links.

TFont.noValues

The example also includes TFont with a noValues reverse derived attribute. The font has 3 style properties (bold, underline, blink). Setting any of these clears noValues, and setting noValues clears all three style properties.
