Transactions

Atomic operations and rollback support

Overview

Bold supports database transactions to ensure data integrity. Multiple changes can be grouped and committed atomically, or rolled back if errors occur.

Features

  • Atomic commits - All changes succeed or none do
  • Rollback - Undo all changes in a transaction
  • Nested transactions - Savepoints within transactions
  • Isolation - Changes invisible until committed

How to Test

  1. Make several changes to objects in the ObjectSpace
  2. Click Commit to save all changes atomically
  3. Or click Rollback to discard all changes
  4. Try causing an error mid-transaction to see automatic rollback
